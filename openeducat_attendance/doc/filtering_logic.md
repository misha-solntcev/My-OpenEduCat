# Логика фильтрации данных для разных типов пользователей

## Общие принципы
Фильтрация данных в модели `op.subject.grades` реализована на уровне методов `_search` и `read`. Это позволяет автоматически применять фильтрацию при любом доступе к данным, независимо от способа их получения (через интерфейс, API или другие методы).

## Группы пользователей
В системе определены три группы пользователей:
1. Студенты (`__custom__.group_op_students`)
2. Преподаватели (`__custom__.group_op_faculty`)
3. Администраторы (все остальные пользователи)

## Логика фильтрации

### Для студентов
1. Проверяется принадлежность текущего пользователя к группе `__custom__.group_op_students`
2. Определяется запись студента, связанная с текущим пользователем через поле `user_id` в модели `op.student`
3. Если запись найдена, применяется фильтр по `student_id`, ограничивая доступ только к записям этого студента
4. Если запись не найдена (пользователь в группе студентов, но не связан со студентом), фильтрация не применяется

### Для преподавателей
1. Проверяется принадлежность текущего пользователя к группе `__custom__.group_op_faculty`
2. Фильтрация не применяется, преподаватели видят все записи

### Для администраторов
1. Проверяется отсутствие принадлежности к группам студентов и преподавателей
2. Фильтрация не применяется, администраторы видят все записи

## Техническая реализация

### Метод _search
Метод `_search` переопределен для применения фильтрации на уровне базы данных. Это наиболее эффективный способ, так как фильтрация происходит на стороне СУБД.

### Метод read
Метод `read` переопределен для дополнительной проверки прав доступа на уровне приложения. Это необходимо для случаев, когда записи могут быть получены другими способами, минуя метод `_search`.

## Особенности реализации
1. Используются методы `env.ref` для получения ссылок на группы пользователей
2. Применяется логирование для отладки и мониторинга работы фильтрации
3. Учитываются возможные ошибки при получении групп и записей студентов
4. Реализована проверка на случай, если пользователь принадлежит к нескольким группам одновременно
