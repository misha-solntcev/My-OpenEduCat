# Kanban-представление оценок по предметам

## Назначение
Kanban-представление модели `op.subject.grades` предоставляет визуальное отображение оценок студентов по предметам с использованием карточек.

## Особенности представления

### Группировка
- По умолчанию записи группируются по классам (`batch_id`)
- Доступна группировка по студентам и предметам через меню "Группировать по"

### Поиск и фильтрация
- Поддерживает поиск по студентам, предметам и классам
- Фильтрация автоматически применяется в зависимости от типа пользователя:
  - Студенты видят только свои оценки
  - Преподаватели и администраторы видят все оценки

### Визуальные элементы карточки

#### Изображение учебника
- Отображается в левой части карточки
- Получается из связанного учебника в модели `op.media` через поле `x_image_128`
- Если учебник с изображением не найден, отображается пустое место

#### Информация в карточке
- Название предмета (жирным шрифтом)
- Имя студента
- Класс
- Средняя оценка

#### Оценки в виде бейджей
- Отображаются в нижней части карточки
- Каждая оценка представлена цветным бейджем:
  - Красный (#fff0f6, #d6336c) для значения 2
  - Желтый (#fff3bf, #997404) для значения 3
  - Зеленый (#ebfbee, #37b24d) для значения 4
  - Синий (#e7f5ff, #1c7ed6) для значения 5
  - Серый (#f8f9fa, #495057) для других значений
- Оценки отображаются жирным шрифтом для лучшей видимости

### Кнопка обновления
- Расположена в верхней части представления
- Доступна только для пользователей с правами менеджера посещаемости
- Вызывает wizard `op.update.subject.grades` для обновления агрегированных данных

## Технические детали
- Использует шаблон QWeb для отображения карточек
- Применяет правильный подход для обработки строк в JavaScript: `('' + mark).trim()` вместо `mark.strip()`
- Отключена возможность создания новых записей (`create="false"`)
- Задан порядок сортировки по умолчанию: `default_order="student_id, subject_id"`
