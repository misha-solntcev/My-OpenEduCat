# Модель op.subject.grades

## Назначение
Модель `op.subject.grades` предназначена для агрегирования данных по оценкам студентов по предметам. Она собирает информацию из записей посещаемости (`op.attendance.line`) и предоставляет удобное представление для отображения в Kanban-виде.

## Поля модели

- `student_id` (Many2one: op.student) - Студент, обязательное поле
- `subject_id` (Many2one: op.subject) - Предмет, обязательное поле
- `batch_id` (Many2one: op.batch) - Класс, обязательное поле
- `marks` (Text) - Все оценки студента по предмету в виде строки, разделенной запятыми
- `average_mark` (Float) - Средняя оценка студента по предмету
- `last_attendance_date` (Date) - Дата последнего занятия
- `total_classes` (Integer) - Общее количество занятий по предмету
- `present_classes` (Integer) - Количество посещенных занятий
- `textbook_image` (Binary, вычисляемое) - Изображение учебника, связанного с предметом

## Методы

### _search
Переопределенный метод поиска, реализующий фильтрацию данных для разных типов пользователей:
- Для студентов: отображаются только записи, относящиеся к этому студенту
- Для преподавателей: отображаются все записи
- Для администраторов: отображаются все записи

### read
Переопределенный метод чтения, реализующий дополнительную проверку прав доступа:
- Для студентов: проверяется, что все читаемые записи принадлежат этому студенту
- Для преподавателей: разрешено чтение всех записей
- Для администраторов: разрешено чтение всех записей

### _compute_textbook_image
Вычисляемое поле, которое находит учебник, связанный с предметом, и получает его изображение из поля `x_image_128` модели `op.media`.

### name_get
Метод, возвращающий строковое представление записи в формате "Студент - Предмет".

## Логика фильтрации
Фильтрация реализована в методах `_search` и `read` на основе принадлежности текущего пользователя к группам:
- `__custom__.group_op_students` - группа студентов
- `__custom__.group_op_faculty` - группа преподавателей

Для студентов проверяется связь пользователя со студентом через поле `user_id` в модели `op.student`.
